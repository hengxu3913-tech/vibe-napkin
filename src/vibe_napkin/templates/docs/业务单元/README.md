# 业务单元 — 编写规范

## 什么是业务单元？

业务单元是一个功能点的**纵向贯穿文档**，把产品描述、技术实现、数据库设计、部署配置都写在一个文件里（≤ 200 行）。

## 标准 6 段式模板

```markdown
# [业务名称]

## 关键词
英文,中文,俄文 逗号分隔，用于向量库召回匹配

## 业务规则
- 规则 1（一句话说明核心逻辑）

## 代码位置
- 函数/类名：`路径::符号` 行号

## 关联业务单元
- [相关单元名](./关联单元.md)

## 历史决策（近 3 次）
- YYYY-MM-DD（commit hash）：决策内容，原因

## 变更触发器
- 什么情况下需要回来改这里
```

## 编写约定

- 每个文件 ≤ 200 行
- `## 关键词` 和 `## 业务规则` 为必填段
- 关联单元使用相对路径引用
- 代码位置使用 `路径::符号 行号` 格式
